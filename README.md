# Task 1
В drawio схеме есть две вкладки: as-is и to-be
- as-is - исходная архитектурная схема 
- to-be - то, к чему стоит прийтич

в Схеме to-be отображено:
- Разделение трафика по географическому расположению
- Отображено решение горизонтального масштабирования подов системы
- Определена стратегия фейловера: Active-StandBy за счет географического разделения - в случае падения кластера в одной из зон, трафик уходит в другой кластер, пока не будут выполнены восстановительные работы
- Отображена конфигурация базы данных в виде репликации данных
   - подход с шардированием, считаю избыточным, так как размер базы данных не превышает 50Гб
